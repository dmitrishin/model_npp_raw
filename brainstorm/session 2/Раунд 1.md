**Идея 1. Вероятностный график лицензирования с GERT-ветвлениями**

Что: Построить GERT-модель процесса лицензирования (B3, B5), где каждый узел — точка решения регулятора (одобрено / замечания / отказ), с вероятностями переходов на основе статистики по проектам-аналогам.

Почему даст эффект: Лицензирование длится от 0,5 до 6 лет, и его неопределённость — главный источник задержек подготовительного периода. Детерминированные графики не отражают итерационную природу процесса (замечания → доработка → повторная подача).

Ожидаемый эффект: Реалистичная оценка сроков лицензирования (P50, P80, P95), выявление узких мест в процессе, обоснование параллельного запуска работ до получения лицензии.

Как применять: GERT-граф с узлами решений (подача заявки, экспертиза ПООБ, ГЭЭ, лицензия на размещение, лицензия на сооружение). На каждом узле — вероятности исходов из базы аналогов. Монте-Карло для расчёта распределения общей длительности. Критерий Гурвица для выбора момента начала параллельных работ.

Этап: Подготовительный период, Этап 1.

**Идея 2. Оптимизация стратегии инвестирования в подготовительный период через Парето-фронт**

Что: Многокритериальная оптимизация объёма вложений в подготовительный период (рекомендация С2) по четырём критериям: T (сроки основного периода), C (общая стоимость проекта), R (риск потери средств при отмене), Q (степень готовности к строительству).

Почему даст эффект: Рекомендация С2 описывает фундаментальное противоречие — больше вложишь в подготовку → ниже риски строительства, но выше потери при отмене. Сейчас это решение принимается экспертно, без формализации.

Ожидаемый эффект: Набор Парето-оптимальных стратегий инвестирования с количественной оценкой trade-off. Обоснованный выбор между агрессивной и консервативной стратегией для конкретного проекта.

Как применять: Определить вектор решений (объём проектирования, сроки закупки ОДЦИ, масштаб подготовки площадки). Для каждой комбинации — Монте-Карло моделирование сценариев (проект идёт / проект отменяется на разных стадиях). Построение Парето-фронта F(Γ) = (T, C, R, Q) → min. Выбор точки на фронте через критерий Гурвица с параметром α, отражающим «аппетит к риску» заказчика.

Этап: Подготовительный период, Этап 1.

**Идея 3. Монте-Карло симуляция критического пути ТМР основного периода**

Что: Стохастическое моделирование критического пути тепломонтажных работ (ТМР) — монтаж трубопроводов, сварка, арматура — которые определяют критический путь графиков строительства АЭС.

Почему даст эффект: ТМР прямо названы в документе как определяющие критический путь. Длительность сварки стыков, контроля, переделок — стохастические величины. Детерминированный CPM не учитывает разброс.

Ожидаемый эффект: Определение вероятностных сроков завершения ТМР (P50, P80), выявление подкритических путей, обоснование размеров буферов и резервов, приоритизация ресурсов.

Как применять: PERT-оценки длительности каждой операции ТМР (a, m, b) на основе нормативов и фактических данных ЗАЭС и других проектов. L = 10000+ реализаций Монте-Карло. Анализ частоты попадания каждого пути в критический. Локальная оптимизация сегментов с наибольшим вкладом в дисперсию общей длительности.

Этап: Основной период.

**Идея 4. GERT-модель процесса контрактации оборудования с деревьями решений**

Что: Моделирование процесса контрактации оборудования (раздел 9 СТО, рекомендация J1) как GERT-сети, где узлы — этапы контрактации (ТЗ → тендер → оценка → контракт → изготовление → приёмка), а дуги имеют вероятности переходов (успех, доработка ТЗ, смена поставщика, брак).

Почему даст эффект: Контрактация ОДЦИ занимает 3–5 лет, и срыв поставки одной позиции блокирует весь монтаж. Процесс итерационный: возможен отказ поставщика, брак при изготовлении, повторный тендер. Это типичная GERT-задача.

Ожидаемый эффект: Определение вероятностных сроков поставки по каждой волне контрактации (6 волн по СТО). Обоснование момента запуска закупок. Идентификация позиций с наибольшим риском срыва.

Как применять: GERT-граф по каждому типу оборудования. Вероятности переходов из статистики прошлых проектов. Деревья решений в узлах выбора (например: менять поставщика или ждать). Критерий Сэвиджа для минимизации «сожаления» при выборе стратегии закупки.

Этап: Подготовительный период (Этап 2) и основной период.

**Идея 5. ML-прогнозирование длительности операций на основе атрибутов**

Что: Обучение моделей XGBoost/CatBoost для прогнозирования длительности строительных и монтажных операций на основе вектора атрибутов: тип работ, объём, квалификация бригады, погодные условия, доступность материалов, номер блока (первый / серийный).

Почему даст эффект: PERT-оценки (a, m, b) назначаются экспертно и одинаково для всех условий. ML-модель учитывает контекст и обучается на фактических данных, давая более точные прогнозы.

Ожидаемый эффект: Повышение точности прогнозирования длительности операций на 20–40% по сравнению с экспертными PERT-оценками. Автоматическая корректировка графика при изменении условий.

Как применять: Сбор данных с предыдущих проектов АЭС (ЗАЭС, Ленинградская, Нововоронежская, экспортные). Формирование обучающей выборки D = {(x_ij, y)}. Обучение ансамблевых моделей. Интеграция в систему календарно-сетевого планирования для автоматического обновления PERT-параметров.

Этап: Основной период (с обучением на данных подготовительного).

**Идея 6. Обнаружение аномалий в ходе строительства методами Isolation Forest / Autoencoder**

Что: Непрерывный мониторинг хода строительства с автоматическим выявлением аномалий — операций, темп которых аномально отклоняется от прогноза.

Почему даст эффект: На строительстве АЭС тысячи параллельных операций. Отклонение одной операции может неочевидным образом повлиять на критический путь через 6–12 месяцев. Человек не может отследить все взаимосвязи.

Ожидаемый эффект: Раннее предупреждение о формирующихся задержках за 3–6 месяцев до их проявления на критическом пути. Сокращение времени реакции руководства.

Как применять: Вектор состояния каждой операции x_i = (плановая длительность, фактический прогресс, расход ресурсов, погода, ...). Isolation Forest или Autoencoder обучаются на «нормальном» ходе работ. Операции с высоким anomaly score → автоматический алерт руководству.

Этап: Основной период.

**Идея 7. Критерий Вальда для выбора проектных решений ядерного острова**

Что: Применение критерия Вальда (максимин) при выборе между альтернативными проектными и строительными решениями для объектов, связанных с ядерной безопасностью — реакторное здание, системы безопасности, защитные оболочки.

Почему даст эффект: Для критически важных решений (safety-related) в статье Дмитришина прямо указано: критерий Вальда подходит для «safety, critical operations, regulatory requirements». Сейчас выбор проектных решений не формализован через теорию принятия решений.

Ожидаемый эффект: Формализация и документирование процесса выбора решений с явным учётом наихудших сценариев. Повышение обоснованности решений перед регулятором.

Как применять: Матрица решений: альтернативы A_i (варианты конструктива), состояния природы S_j (сейсмика, наводнение, комбинации нагрузок), полезности u(A_i, S_j). Критерий Вальда: выбор варианта с наилучшим худшим исходом. Документирование обоснования для ООБ.

Этап: Подготовительный период (проектирование, раздел D).

**Идея 8. Итеративная оптимизация графика пусконаладочных работ (ПНР)**

Что: Применение алгоритма итеративной оптимизации критических путей из статьи к графику ПНР (этапы А-0 → А-4 → Б → В → Г), с пересчётом критического пути после каждой оптимизации сегмента.

Почему даст эффект: ПНР — финальный этап, где накапливаются все предыдущие задержки. Рекомендация G6 требует планирования «от обратного» — от даты ввода через этапы ПНР. Итеративная оптимизация позволяет последовательно сжимать критический путь, переключаясь на подкритические.

Ожидаемый эффект: Сокращение длительности ПНР на 5–15% за счёт перераспределения ресурсов между этапами. Выявление «скрытых» подкритических путей, которые становятся критическими после оптимизации.

Как применять: CPM-модель ПНР с PERT-оценками. Итерация k: оптимизация текущего критического пути P(k) → пересчёт → переход к P(k+1) если D_P(k+1) < D_P(k). Локальная оптимизация каждого сегмента методами динамического программирования. Остановка при достижении P_opt.

Этап: Основной период (планирование ПНР начинается в подготовительном).

**Идея 9. Многомерный предиктивный граф для управления передачей помещений**

Что: Построение многомерного графа событий-времени-атрибутов (раздел 8 статьи) для графика передачи помещений (рекомендация G10) — от строительной готовности через монтаж, ТМР, ЭМР, вентиляцию, отделку до сдачи под ПНР.

Почему даст эффект: Каждое помещение проходит множество состояний с разными атрибутами (тип работ, бригада, доступность, чистота). Граф x_i = (t_i, a_i) позволяет моделировать все возможные траектории готовности помещений с учётом взаимных блокировок.

Ожидаемый эффект: Оптимизация последовательности передачи помещений, минимизация простоев бригад и конфликтов доступа. Сокращение сроков сдачи помещений на 10–20%.

Как применять: Каждое помещение — поток состояний в графе. Атрибуты: строительная готовность (%), монтажная готовность, чистота, давление испытаний, допуск ПНР. Связи e_ij с ограничениями (нельзя начать ЭМР до закрытия коробки). Оптимизация маршрутов через многокритериальный поиск на графе.

Этап: Основной период.

**Идея 10. GNN для прогнозирования рисков в сетевом графике проекта**

Что: Применение графовых нейронных сетей (GCN, GAT) к сетевому графику проекта сооружения АЭС для прогнозирования вероятности задержки каждого узла с учётом топологии графа.

Почему даст эффект: Классический CPM учитывает только прямые связи. GNN агрегирует информацию от соседей по графу и «видит» паттерны распространения задержек через всю сеть — то, что не видит ни CPM, ни обычный ML.

Ожидаемый эффект: Идентификация узлов с высоким риском каскадных задержек, которые не видны при анализе отдельных операций. Приоритизация управленческого внимания.

Как применять: Сетевой график проекта (5000–15000 операций) как вход GNN. Признаки узлов: тип работ, длительность, ресурсы, текущий статус. Обучение на данных Монте-Карло симуляций. Предсказание: P(задержка > X дней) для каждого узла.

Этап: Основной период.

**Идея 11. Критерий Байеса для выбора схемы ценообразования контракта**

Что: Формализация выбора схемы ценообразования генподрядного контракта (рекомендация I2: фиксированная цена / возмещение затрат / целевая цена / гибрид) через критерий Байеса с использованием распределений вероятностей состояний рынка.

Почему даст эффект: Выбор схемы ценообразования — одно из ключевых решений подготовительного периода, определяющее распределение рисков на весь проект. Сейчас выбирается экспертно, без количественной оценки ожидаемой полезности.

Ожидаемый эффект: Количественное обоснование выбора контрактной схемы с учётом вероятностей различных рыночных сценариев. Оптимизация баланса риск/цена.

Как применять: Альтернативы Γ_i = {фикс, cost-plus, target, гибрид}. Состояния природы S_j = {стабильный рынок, инфляция, дефицит ресурсов, санкции}. Вероятности p_j — из макроэкономических прогнозов. Полезности u(Γ_i, S_j) — NPV проекта при каждой комбинации. Γ* = argmax Σ p_j · u(Γ_i, S_j).

Этап: Подготовительный период, Этап 1.

**Идея 12. Кластеризация проектов-аналогов для калибровки модели**

Что: Кластеризация (K-means, DBSCAN) завершённых проектов АЭС по вектору атрибутов (тип реактора, мощность, страна, регуляторная среда, модель контрактации, опыт подрядчика) для определения наиболее релевантного кластера-аналога при начале нового проекта.

Почему даст эффект: Калибровка PERT-оценок и вероятностей GERT-переходов требует данных аналогов. Неправильный выбор аналога (например, использование данных Олкилуото для проекта в Китае) даёт ложные прогнозы. Кластеризация формализует подбор аналогов.

Ожидаемый эффект: Повышение точности начальных оценок модели на 30–50%. Объективный выбор проекта-аналога вместо субъективного.

Как применять: База данных из 50–100 проектов АЭС (IAEA PRIS, NEA). Вектор признаков: 15–20 атрибутов. Кластеризация → определение кластера нового проекта → использование статистик кластера для инициализации PERT-параметров и GERT-вероятностей.

Этап: Подготовительный период, Этап 1 (калибровка модели).

**Идея 13. Локальная оптимизация сегмента «изготовление и поставка ОДЦИ»**

Почему даст эффект: ОДЦИ — позиции с наибольшим вкладом в общую длительность проекта (3–5 лет изготовления). Оптимизация именно этого сегмента даёт максимальное сжатие общего графика.

Ожидаемый эффект: Сокращение длительности цикла ОДЦИ на 3–6 месяцев за счёт оптимального сочетания параллелизации, выбора поставщика и логистики.

**Идея 14. Количественный анализ рисков графика (QRA) с полной интеграцией модели**

Что: Реализация рекомендации H2 (QRA) не как отдельного упражнения, а как полноценного применения всей модели Дмитришина: PERT-оценки → Монте-Карло → идентификация критических путей → GERT для точек решений → критерии выбора → оптимизация.

Почему даст эффект: QRA в текущей практике — это обычно просто Монте-Карло поверх CPM. Модель Дмитришина добавляет управляемость: не только «какова вероятность задержки», но и «что оптимально сделать, чтобы её уменьшить».

Ожидаемый эффект: Переход от пассивного анализа рисков к активному управлению. Обоснование размеров резервов (contingency) на уровне P80. Обоснование управленческих решений по ускорению.

Как применять: CPM-модель ИКГ (уровень 2) → PERT-оценки всех операций → Монте-Карло (L=10000) → выявление критических путей → GERT-моделирование в точках решений → локальная оптимизация наиболее чувствительных сегментов → итеративная оптимизация → финальный прогноз с доверительными интервалами.

Этап: Подготовительный период (разработка) и основной период (применение).

**Идея 15. Адаптивное планирование методом «от обратного» с ML-коррекцией**

Что: Реализация рекомендации G6 (планирование от даты ввода через этапы ПНР) с помощью модели: сначала ML прогнозирует длительность каждого этапа ПНР на основе атрибутов проекта, затем обратный расчёт определяет необходимые даты начала предшествующих работ, а Монте-Карло даёт доверительные интервалы.

Почему даст эффект: Планирование «от обратного» требует точных оценок длительности ПНР, которые сильно зависят от конкретного проекта. ML позволяет учесть эту зависимость, а не использовать средние по отрасли.

Ожидаемый эффект: Более реалистичные директивные сроки. Обоснованный момент начала каждого этапа. Раннее выявление нереалистичности целевой даты ввода.

Как применять: ML-модель обучается на данных ПНР завершённых блоков (длительности этапов А-0…Г-2 + атрибуты блока). Для нового проекта прогнозируются длительности → обратный расчёт → PERT + Монте-Карло для вероятностных оценок → итеративная оптимизация.

Этап: Подготовительный период (планирование), основной период (коррекция).

### 1. Полная GERT/PERT + Monte Carlo модель всего подготовительного периода для прогнозирования даты «Первого бетона»
**Что предлагается**: Построить DAG/GERT-граф всех ключевых событий этапов 1–2 подготовительного периода (A1–A5 госполитика, B1–B5 лицензирование/ОВОС/площадка, K1–K7 инженерная подготовка + СМБ/пионерная база, J1 ОДЦИ-контрактация, N1 Readiness Review).  

**Почему даст эффект**: Подготовительный период 2–8 лет — главный источник фатальных задержек (лицензирование 1,5–6+ лет, регуляторные пересмотры, площадочные работы); без вероятностной модели график остаётся детерминистическим и не учитывает ветвления (как в примерах VC Summer 4 года на COL, Olkiluoto непрерывные замечания STUK).  

**Ожидаемый эффект**: 80–90 % вероятность ПБ в срок ±6–12 мес., выявление 80/20 critical/subcritical paths, contingency-буферы, сокращение подготовительного на 6–18 мес. в среднем.  

**Как применять модель**: Beta/normal распределения Tij из исторических данных (ЗАЭС, Аккую, СТО); 10 000 MC-симуляций; GERT для ветвлений (регуляторные замечания → rework loop); XGBoost/GNN для обновления параметров по реальным progress; итеративная оптимизация critical paths.  

**Этап**: Подготовительный (оба этапа, особенно перед FID и N1).  

### 2. Многомерная Pareto-оптимизация стратегии инвестирования и контрактной модели (C1–C3 + I1–I7)
**Что предлагается**: Моделировать альтернативы (EPC full vs split, early vs JIT контрактация, госгарантии vs частное финансирование) как trajectories в multidimensional event-time graph с вектором F(Γ) = [T, C, R, Q].  

**Почему даст эффект**: Ранние вложения в подготовку (лицензирование, ОДЦИ, СМБ) снижают риски основного периода, но создают sunk costs; неправильное распределение рисков — главная причина overruns (NEA 7530, примеры Flamanville/Sanmen).  

**Ожидаемый эффект**: Снижение стоимости капитала в 1,5–2 раза, оптимальное распределение «средств под риском» по gates, -15–25 % общих затрат проекта.  

**Как применять модель**: Scalarization αT + βC + γR + δQ или Pareto-front; GERT decision trees на инвестиционных воротах; Bayes/Hurwicz/Savage/Wald критерии по сценариям (политические изменения, регуляторные shifts); ML (CatBoost) для эмпирических utility.  

**Этап**: Подготовительный (до EPC-контракта и FID).  

### 3. GERT + ML для оптимизации цепочки поставок ОДЦИ и локализации (J1–J4)
**Что предлагается**: Многомерный predictive graph для поставок long-lead items (корпус реактора, ГЦНА, парогенераторы) с атрибутами (время, стоимость, риск импортных ограничений, % локализации).  

**Почему даст эффект**: ОДЦИ — классический critical path; задержки 6–18 мес. типичны, локализация/экспортный контроль — ключевой риск экспортных проектов (J4, IAEA).  

**Ожидаемый эффект**: Just-in-time поставки с 85 % вероятностью, -15–25 % стоимости логистики/складов, избежание cascade delays в ТМР.  

**Как применять модель**: PERT beta-dist + Monte Carlo; GNN/Temporal Graph Networks для multi-supplier graph; local optimization сегментов (выбор поставщика/локализации); XGBoost для предсказания задержек по историческим данным (Росатом + международные).  

**Этап**: Подготовительный (опережающая контрактация) + ранний основной.  

### 4. Локальная + итеративная оптимизация сегментов сооружения СМБ/пионерной базы и ВЗиС (K2–K7 + 13/25 СТО)
**Что предлагается**: Разбить работы по 1–4 пусковым комплексам СМБ (арматурное хозяйство, БРХ, арматурно-сварочный, временные дороги, водопонижение) на сегменты и оптимизировать последовательность/ресурсы.  

**Почему даст эффект**: Без timely СМБ невозможно поточное строительство с шагом 12–18 мес. (ОТП-86, ЗАЭС опыт); задержки базы = простой всего проекта.  

**Ожидаемый эффект**: Завершение инфраструктуры на 4–8 мес. раньше, +15–25 % эффективности потока, оптимальная пиковая численность 3500 без простоев/перерасходов на жильё.  

**Как применять модель**: PERT-сегмент + local LP/DP min x·(αμ + βC + γR); iterative recalculate critical paths с переходом на subcritical; GNN для коллизий в 3D-модели; Monte Carlo на ресурсных сценариях (погода, грунты).  

**Этап**: Подготовительный (Этап 2).  

### 5. Многокритериальная оценка Construction Readiness Review (N1) перед Первым бетоном
**Что предлагается**: Количественный шлюз готовности (зрелость ПСД, поставки, графики, лицензии, персонал, СМБ) как multidimensional scoring + probabilistic simulation.  

**Почему даст эффект**: SRS-119/NEA 7530/Рекомендации N1 требуют формального review; старт «сырого» проекта — главная причина overruns (Olkiluoto, VC Summer).  

**Ожидаемый эффект**: Объективный go/no-go с quantified risk (>95 % вероятность успеха), план остаточных работ, снижение рисков основного периода на 20–30 %.  

**Как применять модель**: Multidimensional xi = (ti, готовность атрибуты); Monte Carlo сценариев незавершённости; Wald/Savage для worst-case resilience; GNN/Autoencoder для anomaly detection в данных; Pareto для trade-off.  

**Этап**: Конец подготовительного (прямо перед ПБ).  

### 6. ML + Lessons Learned для адаптивного прогнозирования и оптимизации ТМР (критический путь основного периода)
**Что предлагается**: Гибрид GERT + GNN/Temporal Graph Networks для тепломонтажных работ (сварка ГЦТ, укрупнение блоков трубопроводов) с learning curve от предыдущих блоков/проектов.  

**Почему даст эффект**: ТМР — главный critical path (Construction.md); эффект обучения в серии (O1) даёт 15–20 % экономии на 2-м блоке, 30–40 % на 6+.  

**Ожидаемый эффект**: Сокращение основного периода на 3–8 мес., шаг ввода блоков 12 мес., перенос уроков между проектами.  

**Как применять модель**: Итеративная оптимизация critical paths; ML (CatBoost/GNN) на historical data (ЗАЭС 5 блоков за 9 лет) + real-time update; multidimensional optimization (T + C + R + Q).  

**Этап**: Основной (с заделом в подготовительном для серийности).  

### 7. GERT + decision criteria для управления регуляторными и политическими рисками (A1–A5, B1–B5)
**Что предлагается**: Ветвления в GERT на узлах «изменение требований регулятора/политики» (post-Fukushima, новые интерпретации ASN/NRC) с критериями принятия решений.  

**Почему даст эффект**: Регуляторные/политические shifts — топ-причина задержек/перерасходов (Flamanville 4 года на интерпретацию, Sanmen 54 мес. после 2011).  

**Ожидаемый эффект**: Проактивные contingency-планы, снижение времени на доработки на 1–3 года, повышенная resilience.  

**Как применять модель**: Decision trees на branching nodes; Wald (worst-case) для safety, Savage (min regret) для политики, Bayes/Hurwicz для баланса; ML (XGBoost) для вероятностей pj по историческим данным.  

**Этап**: Подготовительный (особенно экспорт).  

### 8. Multidimensional optimization + ML для персонала, соц. инфраструктуры и серийности (L1–L4 + O1)
**Что предлагается**: Моделировать график потребности персонала (пиковая 3500), обучение, жильё и эффект обучения в серии как event-time graph с атрибутами (культурные/законодательные особенности L2).  

**Почему даст эффект**: На экспорте/удалённых площадках дефицит кадров и спекулятивный рост аренды — критический риск (L2–L4, СТО); серийность даёт 15–30 % экономии.  

**Ожидаемый эффект**: Стабильный коллектив, -10–15 % overhead на соц.инфраструктуру, точное планирование шага ввода блоков.  

**Как применять модель**: Monte Carlo на графике движения ресурсов; CatBoost clustering сценариев (климат, локализация); Pareto optimization размещения жилья; iterative learning curve для последующих блоков.  

**Этап**: Подготовительный (опережающее строительство жилья/УПК) + программный для серии.  

### 9. Интеграция с BIM/PLM/PIM в multidimensional predictive graph (M1–M2 + D1–D2 + F1–F4)
**Что предлагается**: Загружать 3D-модель + PIM/K-PIM + графики в event-time graph; GNN для clash detection и оптимизации последовательности выпуска РД/монтажа.  

**Почему даст эффект**: Коллизии и rework — major cost driver (до 20–30 %); рекомендации M/D подчёркивают цифровую подготовку и проверку коллизий до начала СМР.  

**Ожидаемый эффект**: -20–40 % rework, рост заводской готовности до 40 %, модульность, ускорение первоочередной РД.  

**Как применять модель**: xi включает spatial coordinates + t; Temporal Graph Networks/GNN для prediction коллизий; Pareto optimization последовательностей; Autoencoder для anomaly в изменениях.  

**Этап**: Подготовительный (проектирование + РД) + основной.  

### 10. Гибридная модель QRA + anomaly detection + Lessons Learned для всего жизненного цикла (H1–H3 + F5)
**Что предлагается**: Полная интегрированная система рисков с Isolation Forest/Autoencoder на event-time graph + автоматическое извлечение уроков для обновления модели.  

**Почему даст эффект**: Рекомендации H/F подчёркивают необходимость proactive risk management и извлечения уроков; модель позволяет переходить от статического к адаптивному управлению (особенно на экспорте).  

**Ожидаемый эффект**: Снижение variance проекта на 30–50 %, минимизация повторяющихся ошибок, real-time recommendations.  

**Как применять модель**: Monte Carlo QRA на всей сети; ML anomaly detection на deviations; clustering + transfer learning из past проектов (ЗАЭС, международные); GERT для rework loops.  

**Этап**: Все этапы (с акцентом на подготовительный для early warning).

**Идея №1. Прогнозирование сроков получения Лицензии на сооружение (Лицензирование).**
*   **Что предлагается:** Использовать PERT и Монте-Карло для вероятностной оценки длительности этапов лицензирования (экспертиза в НТЦ ЯРБ, Ростехнадзор, получение заключений). Вместо фиксированной даты «за 1.5 года до ПБ» строить распределение вероятностей.
*   **Почему это даст эффект:** Рекомендации (раздел B5) указывают, что цикл лицензирования в РФ жестко регламентирован, но документально показывает разброс от 3 до 9 лет в международной практике. Фиксированные графики часто срываются из-за задержек в ответах на вопросы регулятора. Модель позволит рассчитать буфер времени и не планировать «Первый бетон» при низких шансах на лицензию.
*   **Ожидаемый эффект:** Снижение риска простоя стройплощадки из-за отсутствия лицензии. Оптимизация страховых резервов времени.
*   **Как применять модель:** Для каждого этапа (подача заявки, экспертиза, доработка) задать бета-распределение длительностей на основе исторических данных по предыдущим блокам. Провести 1000 итераций Монте-Карло. Получить P90 (90% вероятность) срока завершения.
*   **Этап:** Подготовительный период.

**Идея №2. Оптимизация стратегии финансирования (Критерий Гурвица).**
*   **Что предлагается:** Использовать критерий Гурвица из модели для выбора стратегии финансирования подготовительного периода: «Максимальные вложения в проектную документацию (ПД) до ПБ» против «Минимизация вложений до инвестиционного решения».
*   **Почему это даст эффект:** В разделе С2 Рекомендаций описана дилемма: вкладывать деньги в ПД и рисковать потерять их при отмене проекта, или экономить и получать перерасход в основном периоде. Участники часто выбирают эмоционально. Формальный критерий позволит обосновать выбор.
*   **Ожидаемый эффект:** Выбор оптимального баланса «риск-стоимость». Снижение стоимости капитала за счет обоснованности решений для инвесторов/банков.
*   **Как применять модель:** Построить матрицу решений (инвестировать много / инвестировать мало) и состояний природы (проект одобрен / проект отменен). Задать весовой коэффициент пессимизма α (например, 0.7 для консервативного сценария). Рассчитать функцию полезности по формуле из статьи.
*   **Этап:** Подготовительный период (до принятия инвестиционного решения).

**Иdea №3. Управление критическим путем монтажа трубопроводов (ТМР) с помощью ML.**
*   **Что предлагается:** Применить методы машинного обучения (XGBoost/CatBoost) для прогнозирования производительности сварочных работ и монтажа трубопроводов на основе погодных условий, квалификации персонала, наличия материалов и сложности узлов.
*   **Почему это даст эффект:** В Справочнике указано, что тепломонтажные работы (ТМР) и сварка определяют критический путь основного периода. В модели Дмитришина предлагается переход от теоретических beta-распределений к эмпирическим при наличии данных (>30 наблюдений). Накопленные данные со стройплощадок позволят точнее прогнозировать сроки.
*   **Ожидаемый эффект:** Сокращение длительности основного периода строительства на 2-3 месяца за счет своевременного выявления отставаний и перераспределения ресурсов.
*   **Как применять модель:** Обучить модель на исторических данных суточной выработки бригад (погонных метров шва, тонн смонтированных труб). Вносить текущие данные в модель для корректировки прогноза завершения этапа.
*   **Этап:** Основной период.

**Идея №4. GERT-модель для принятия решения о «заморозке» проектирования (Design Freeze).**
*   **Что предлагается:** Использовать GERT-сеть для моделирования процесса завершения проектирования. Узел ветвления: «Проект готов к выпуску РД» или «Требуется доработка от поставщика оборудования».
*   **Почему это даст эффект:** Рекомендации (раздел D1) настаивают на завершенности проекта к моменту ПБ. Но на практике поставщики ОДЦИ часто задерживают данные. Обычный PERT не учитывает циклы «переделок» (loops). GERT позволяет моделировать итерации.
*   **Ожидаемый эффект:** Реалистичная оценка готовности РД. Предотвращение начала строительства с незавершенным проектом (главный фактор удорожания по NEA 7530).
*   **Как применять модель:** Построить стохастическую сеть, где вероятность возврата на доработку рассчитывается на основе надежности конкретного поставщика. Рассчитать математическое ожидание времени готовности полного комплекта документации.
*   **Этап:** Подготовительный период.

**Идея №5. Локальная оптимизация графика поставок оборудования (Just-in-Time).**
*   **Что предлагается:** Использовать локальную оптимизацию из модели для минимизации функции затрат на хранение и простой монтажа. Целевая функция: min(Затраты_на_хранение + Штрафы_за_простой).
*   **Почему это даст эффект:** СТО требует поставок по принципу «Just-in-time», но АЭС — сложный объект, и опоздание даже одного блока останавливает монтаж. Модель Дмитришина предлагает методы оптимизации в многомерном пространстве атрибутов.
*   **Ожидаемый эффект:** Сокращение площадей складирования на СМБ, уменьшение иммобилизации оборотных средств.
*   **Как применять модель:** Разбить график поставок на сегменты (комплекты). Для каждого сегмента решить задачу оптимизации, связывающую время, стоимость хранения и риск опоздания транспорта.
*   **Этап:** Основной период (подготовка и монтаж).

**Идея №6. Выбор конфигурации АЭС методом скаляризации вектора целей.**
*   **Что предлагается:** При выборе проекта АЭС (Рекомендации B2) использовать метод скаляризации векторной целевой функции F(Γ) = [T, C, R, Q] (Время, Стоимость, Риск, Качество/Безопасность).
*   **Почему это даст эффект:** Часто выбор идет между «быстро и дорого» или «дешево и медленно». Модель позволяет свести 4 критерия к одному обобщенному показателю с весами (α, β, γ, δ), что важно для обоснования перед инвесторами.
*   **Ожидаемый эффект:** Прозрачный и формализованный выбор технологического решения (например, выбор между ВВЭР-1200 и ВВЭР-ТОИ для конкретной площадки).
*   **Как применять модель:** Для каждого варианта проекта рассчитать вектор характеристик. Задать веса экспертно или на основе политики компании. Выбрать вариант с min(F).
*   **Этап:** Подготовительный период (предпроектная стадия).

**Идея №7. Детекция аномалий в ходе строительства (Autoencoder).**
*   **Что предлагается:** Использовать методы машинного обучения (Autoencoder, Isolation Forest) для автоматического выявления аномалий в отчетах о ходе работ.
*   **Почему это даст эффект:** При огромном объеме работ на АЭС менеджеры могут пропустить критическое отставание на второстепенном участке, которое «вдруг» становится критическим. Модель из статьи (раздел 9) предлагает методы поиска аномалий.
*   **Ожидаемый эффект:** Раннее предупреждение о проблемах (раньше, чем они станут критическими). Снижение риска срыва ключевых событий.
*   **Как применять модель:** Подать на вход модели векторы состояния объектов (выполнение плана, расход ресурсов). Модель сама выявит объекты, чье поведение отклоняется от «нормального» сценария.
*   **Этап:** Основной период.

**Идея №8. Моделирование сценариев пусконаладочных работ (Этап А-Б-В-Г).**
*   **Что предлагается:** Использовать критерий Вальда (Maximin) для планирования сроков ПНР.
*   **Почему это даст эффект:** Пусконаладка — финальный рывок. Здесь максимально проявляются «скрытые дефекты». Критерий Вальда ориентируется на худший случай, что идеально подходит для планирования резерва времени перед физическим пуском.
*   **Ожидаемый эффект:** Гарантированный срок ввода в эксплуатацию (без переносов даты пуска, что критично для репутации и штрафов).
*   **Как применять модель:** Рассмотреть альтернативные графики ПНР. Выбрать тот, который дает лучший результат при самом плохом сценарии (например, при обнаружении дефекта в ГЦН).
*   **Этап:** Основной период (конец).
