## Группа А. ПЛАНИРОВАНИЕ ГРАФИКОВ И QRA

### А1. Вероятностный ИКГ уровня 2 вместо детерминированного CPM
**Где:** ИКГ уровня 2 (Рекомендации, G1; Приложение 1, раздел 12)
**Что:** PERT с бета-распределениями → Монте-Карло → P50/P80/P90. ML (XGBoost) предсказывает распределения длительностей по реальному опыту, очищая от «когнитивного оптимизма». Критерии Вальда/Гурвица для выбора целевой даты.
**Зачем:** Обоснованные резервы вместо политически мотивированных дат.
**Почему это сработает:** Полный цикл CPM + QRA + критерии решений, отсутствующий в стандартных инструментах.

### А2. Итеративная оптимизация критических путей (прескриптивный QRA)
**Где:** QRA (Рекомендации, H2; Приложение 1, раздел 12)
**Что:** PERT → Монте-Карло → выявление критических/субкритических путей → локальная оптимизация → пересчёт → итерация до P_opt. Создание robust schedule с буферами в точках максимального риска.
**Зачем:** От диагностики к прескрипции: какие работы ускорить, где заложить буферы.
**Почему это сработает:** Итеративная оптимизация (раздел 7) — ключевое отличие от стандартного QRA.

### А3. Обоснование продолжительности основного периода и шага ввода блоков
**Где:** Варианты 48–96 мес.; шаг ввода 12/18/24 мес. (Рекомендации, G1)
**Что:** Для каждого варианта — полная вероятностная модель + Парето-фронт {T, C, R, Q}.
**Зачем:** Количественное обоснование стратегического решения.

### А4. Парето-фронт «сроки — стоимость — риск — качество»
**Где:** Финансовые риски (С3); оценка стоимости (G2)
**Что:** Многокритериальная оптимизация F(Γ) = {T, C, R, Q} → min. Визуализация Парето-фронта; пересчёт при уточнении стоимости (от класса 5 к классу 3).
**Зачем:** Карта компромиссов вместо одной точки «срок X, стоимость Y».

### А5. Детектор рассинхронизации уровней графиков (3/4)
**Где:** Синхронизация графиков (Рекомендации, G12, G13)
**Что:** Прогнозирование отрыва графика 4 уровня от 3 уровня; автоматическая перестройка.
**Зачем:** Повысить управляемость, сократить фиктивную отчётность.

### А6. Обратное планирование от целевой даты пуска через этапы ПНР
**Где:** Планирование «от обратного» (Рекомендации, G6, D7; Construction.md, этапы А–Г)
**Что:** Вероятностный граф в обратном направлении: от коммерческой эксплуатации к А-0.
**Зачем:** Исключение ситуации «стройка завершена, но блок не готов к вводу».

### А7. [R2] Мост между стратегическим и оперативным планированием (3 года ↔ 6 мес.)
**Где:** Тематическое планирование на 3 года (Рекомендации, G11–G12)
**Что:** Модель как «мост» между уровнями: автоматическая детекция, какие решения уровня 2 уже не выполнимы на уровне 4. Критерий Байеса: ежеквартальное обновление 3-летнего прогноза на основе факта. Замена устаревших бета-распределений эмпирическими.
**Зачем:** Решает проблему G1 о графиках, «теряющих актуальность», создаёт механизм обратной связи факт → прогноз.
**Почему это сработает:** Bayesian updating — математически строгий способ обновления долгосрочного прогноза. Модель адаптируется, а не «ломается» при отклонениях.

### А8. [R2] Единая GERT-модель подготовительного периода (лицензирование + проектирование + контрактация)
**Где:** Комбинация B3–B5 + D1/D9 + J1/I (Приложение 1, разделы 6, 9, 18)
**Что:** Три параллельных потока с узлами сопряжения: ИДЗ от поставщиков → РД → экспертиза → замечания → доработка → новые ИДЗ. Монте-Карло на полной модели даёт распределение даты «Первый бетон» с учётом каскадных взаимовлияний.
**Зачем:** Отдельное моделирование лицензирования, контрактации и проектирования не ловит каскады. Единая модель их ловит.
**Почему это сработает:** Интегрированная модель Дмитришина создана для объединения разнородных процессов в едином графе.

## Группа Б. ТЕПЛОМОНТАЖНЫЕ РАБОТЫ И КРИТИЧЕСКИЙ ПУТЬ

### Б1. GERT-модель сварки трубопроводов с петлями переделок
**Где:** ТМР — критический путь (Construction.md)
**Что:** GERT: сварка → NDE → годен/ремонт/перерезка. Оптимизация технологии сварки, последовательности монтажа, стратегии контроля.
**Зачем:** Каждый дефект — петля переделки. Потенциал сокращения на недели-месяцы.

### Б2. Динамическая оптимизация критического пути в ходе строительства
**Где:** Основной период (Construction.md; Рекомендации, G)
**Что:** Упреждающее перераспределение ресурсов при миграции критического пути (ТМР → электромонтаж → строительные).
**Зачем:** Сокращение основного периода на 3–8 мес.

### Б3. [R2] Расширение GERT ТМР: изоляция и химзащита как дополнительные дуги
**Где:** Антикоррозионная обработка, теплоизоляция трубопроводов (Construction.md)
**Что:** Последовательность: сварка → НК → антикор → теплоизоляция → сдача под ПНР. Оптимизация размера бригад изолировщиков как функция от темпа сварки/НК.
**Зачем:** Системы нельзя сдать под ПНР без изоляции — это «хвост» ТМР, часто забываемый при планировании.
**Почему это сработает:** Расширение GERT-модели дополнительными дугами масштабируется линейно.

### Б4. [R2] Предиктивное качество сварки в реальном времени
**Где:** Контроль качества (Приложение 1, разделы 4.5, 15.4; Рекомендации, L1)
**Что:** ML-классификация: квалификация сварщика + усталость + погода + тип материала + параметры режима → прогноз вероятности дефекта в реальном времени. Автоматическая остановка при высоком риске.
**Зачем:** Снижение брака и переделок на критическом пути. Повышение культуры безопасности.
**Почему это сработает:** ML на исторических данных дефектов выявляет неблагоприятные сочетания факторов раньше человека.

**Зачем:** Турбинный остров — второй по значимости объект. Выставление линии вала — прецизионная операция с непредсказуемыми задержками. Скрытое «узкое горлышко» перед энергопуском.

**Где:** Вентиляционные работы (Construction.md; Рекомендации, G10)

Парето-оптимизация глубины укрупнения на СМБ vs. сроки сдачи. ML-кластеризация помещений по типам вентсистем для оптимизации маршрутов монтажа.

**Что:** Модель потока: строительство → монтаж → электрика → вентиляция → сантехника → отделка → сдача. ML-прогноз узких мест. Локальная оптимизация: распределение отделочных бригад по критерию максимизации сданных помещений к ключевым датам. Кластеризация помещений по типам отделки.
**Зачем:** Отделка — «длинный хвост», блокирующий ПНР. Конфликт доступа с электрикой и КИП.
**Почему это сработает:** Задача назначения бригад на помещения — целочисленная оптимизация (раздел 6).

### Х3. Пространственно-временные коллизии смежных специальностей (GNN)
**Где:** Электромонтаж + вентиляция + ТМР в замкнутых объёмах (Construction.md; Рекомендации, D2)
**Что:** GNN на топологическом графе помещения для предсказания коллизий доступа между специальностями. Строгая очерёдность: воздуховоды → трубы → кабели → изоляция → отделка.

## Группа Ю. [R2] ОПЕРАЦИОННОЕ ПЕРЕПЛАНИРОВАНИЕ («ЦИФРОВОЙ ПРОРАБ»)

### Ю1. Ежедневная переоптимизация заданий бригадам
**Где:** Операционный уровень (Рекомендации, G12; Construction.md)
**Что:** «Цифровой ИИ-прораб»: ежедневный ввод факта → автоматический пересчёт PERT/Монте-Карло на 2–4 недели → выявление новых критических работ → рекомендации по перераспределению бригад. При сбое (нет материалов, сломалась техника) → мгновенный пересчёт локального графа → альтернативное задание бригаде. ML (Temporal Graph Networks) прогнозирует завтрашние задержки по сегодняшнему паттерну. Критерий Байеса для обновления вероятностей.
**Зачем:** Переход от месячных корректировок к проактивному управлению. Выдача 100% выполнимых сменно-суточных заданий. Ликвидация скрытых простоев.
**Почему это сработает:** Локальная оптимизация (раздел 6) на горизонте 1–3 дней — за секунды. Модель Дмитришина изначально заточена на динамический пересчёт путей.

### Ю2. Оптимальный выбор «исполнимых» фронтов на неделю
**Где:** Планирование строительных потоков (Рекомендации, G11–G12)
**Что:** Еженедельный математический выбор набора реально исполнимых фронтов: максимальный совокупный эффект по T/C/R, а не «все доступные работы».
**Зачем:** Снижение распыления ресурсов и числа фронтов, открытых «для вида».

### Ю3. Ежедневная оценка рисков срыва плана (ML)
**Где:** Управление рисками на операционном уровне (Рекомендации, H1)
**Что:** Random Forest: ежедневная оценка P(срыв недельного плана) с учётом погоды, ресурсов, статуса смежных работ, усталости персонала.
**Зачем:** Раннее предупреждение за 3–5 дней, время на превентивные меры.
