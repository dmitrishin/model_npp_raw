## Группа А. ПЛАНИРОВАНИЕ ГРАФИКОВ И QRA

### А1. Вероятностный ИКГ уровня 2 вместо детерминированного CPM
PERT с бета-распределениями → Монте-Карло → P50/P80/P90. ML предсказывает распределения по реальному опыту. Критерии Вальда/Гурвица для выбора целевой даты.

### А2. Итеративная оптимизация критических путей (прескриптивный QRA)
PERT → Монте-Карло → выявление критических путей → локальная оптимизация → пересчёт → итерация до P_opt. Robust schedule с буферами.

### А3. Обоснование продолжительности основного периода и шага ввода блоков
Для каждого варианта (48–96 мес.; шаг 12/18/24 мес.) — полная вероятностная модель + Парето-фронт {T, C, R, Q}.

### А4. Парето-фронт «сроки — стоимость — риск — качество»
Многокритериальная оптимизация F(Γ) = {T, C, R, Q} → min. Визуализация Парето-фронта.

### А5. Детектор рассинхронизации уровней графиков (3/4)
Прогнозирование отрыва графика 4 уровня от 3 уровня; автоматическая перестройка.

### А6. Обратное планирование от целевой даты пуска через этапы ПНР
Вероятностный граф в обратном направлении: от коммерческой эксплуатации к А-0.

### А7. [R2] Мост между стратегическим и оперативным планированием (3 года ↔ 6 мес.)
Байесовское обновление 3-летнего прогноза на основе факта. Детекция невыполнимых решений уровня 2.

### А8. [R2] Единая GERT-модель подготовительного периода
Три параллельных потока (лицензирование + проектирование + контрактация) с узлами сопряжения.

### Б4. [R2] Предиктивное качество сварки в реальном времени
ML-классификация: квалификация сварщика + усталость + погода + материал → прогноз дефекта.

### Б5. [R3] Сквозная прослеживаемость качества сварки

## Группа ЯБ. [R3] ЯДЕРНАЯ БЕЗОПАСНОСТЬ НА ЭТАПЕ СТРОИТЕЛЬСТВА

### ЯБ1. Вероятностная модель взаимодействия с регулятором в ходе строительства
GERT-модель: инспекция → несоответствие/нет → предписание → устранение → проверка → подтверждение/повтор. ML-прогноз «плотности» инспекций. Детектор аномалий: рост частоты нарушений → предупреждение о системной проблеме. Оптимизация расписания Hold Points.

### ЯБ2. Предиктивный индекс культуры безопасности
NLP + ML (Random Forest / Isolation Forest) на ежедневных отчётах и near-miss. Датчики на касках для локализации проблемных зон. Критерий Вальда для превентивной остановки участка при падении индекса.

### ЯБ3. Совмещённая модель строительных работ и лицензионных требований безопасности
Многомерный граф: каждое событие → набор требований безопасности. Автоматическая генерация предупреждений при нарушении gate conditions.

### ЯБ4. Маршрутизация регуляторных взаимодействий
Модель пропускной способности регулятора: что эскалировать немедленно, что пакетировать, что закрывать внутри. Оптимизация очереди вопросов.

### ИН1. Граф интерфейсов между участниками проекта
GNN на графе организаций: детекция «узлов напряжения» с высокой центральностью и низкой пропускной способностью. SNA для выявления критических субподрядчиков. Isolation Forest на потоке RFI/трансмитталов.

### ИН2. Модель интерфейса «строительство — эксплуатация» (передача систем)
GERT: завершение монтажа → испытания → акты → замечания → доработка → приёмка. Многомерное условие готовности (не дата, а совокупность: поставка + монтаж + исп. документация + ЗИП + калибровка).

## Группа ОП. [R3] ОБУЧЕНИЕ ПЕРСОНАЛА И ТРЕНАЖЁРЫ

### ОП1. Синхронизация обучения эксплуатационного персонала с графиком ПНР
PERT-модель: набор → теория → стажировка на аналоге → ПМТ → допуск. Gate conditions к А-1 и Б-1. Монте-Карло: риск неготовности с учётом текучести и провалов на экзаменах. Критерий Гурвица: нехватка персонала vs. перерасход ФОТ.

### ОП2. Полномасштабный тренажёр (ПМТ) как отдельный подграф
GERT: разработка матмоделей → программирование → верификация/валидация → поставка → монтаж → сертификация регулятором. Зависимость: данные для верификации → от ХГО (А-3), но обучение → до Б-1. Критерий Вальда.

### ОП3. Transfer learning между блоками для тренажёрных программ
Данные реальных испытаний блока 1 → калибровка матмодели ПМТ блока 2.

### ОП4. ML-оценка влияния объёма тренажёрной подготовки на длительность ПНР
Регрессия: часы тренировок × тип операции → сокращение длительности этапов Б и В.
