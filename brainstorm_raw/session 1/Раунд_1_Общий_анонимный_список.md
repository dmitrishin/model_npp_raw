## Группа А. ПЛАНИРОВАНИЕ ГРАФИКОВ И КОЛИЧЕСТВЕННЫЙ АНАЛИЗ РИСКОВ (QRA)

### А1. Вероятностный ИКГ уровня 2 вместо детерминированного CPM
**Где:** Планирование сроков — разработка ИКГ уровня 2 (Рекомендации, G1; Приложение 1, раздел 12)
**Что:** Заменить детерминированный CPM-график вероятностной PERT-моделью с бета-распределениями длительностей. Монте-Карло генерирует распределение сроков: P50, P80, P90 для каждого ключевого события. ML (XGBoost) предсказывает распределения длительностей на основе реального опыта, очищая от «когнитивного оптимизма». Критерии Вальда/Гурвица — для выбора целевой даты в зависимости от рискового аппетита заказчика.
**Зачем:** Обоснованные резервы вместо политически мотивированных дат. Реалистичная дата «Первого бетона» и энергопуска. Снижение риска каскадных организационных коллизий.
**Почему это сработает:** Модель формализует задачу, которую ставит G1: CPM + QRA. ML очистит оценки от субъективного оптимизма. PERT + Монте-Карло + критерии решений — полный цикл, отсутствующий в стандартных инструментах.

### А2. Итеративная оптимизация критических путей (прескриптивный QRA)
**Где:** Количественный анализ рисков (Рекомендации, H2; Приложение 1, раздел 12)
**Что:** Полный цикл: PERT → Монте-Карло → выявление критических и субкритических путей → локальная оптимизация сегментов критического пути → пересчёт → переход к следующему критическому пути → итерация до нахождения условно-оптимального плана. Создание устойчивого к возмущениям графика (robust schedule) с буферами в точках максимальной вероятности срыва.
**Зачем:** Переход от диагностики (стандартный QRA даёт только распределение) к прескриптивному результату: какие именно работы ускорить, на сколько, и где заложить буферы. Снижение overruns на 10–25%.
**Почему это сработает:** Итеративная оптимизация критических путей (раздел 7 статьи) — ключевое отличие от стандартного QRA. При каждой итерации субкритический путь может стать критическим — модель это учитывает.

### А3. Обоснование продолжительности основного периода и шага ввода блоков
**Где:** Выбор из вариантов 48, 54, 65, 72, 84, 96 месяцев основного периода; шаг ввода 12/18/24 мес. (Рекомендации, G1)
**Что:** Для каждого варианта построить полную вероятностную модель (PERT + Монте-Карло). Многокритериальная оптимизация: F(Γ) = {T, C, R, Q} → min. Результат — Парето-фронт «продолжительность / стоимость / риск» для обоснования стратегического решения.
**Зачем:** G1 требует технико-экономическое обоснование этих параметров, определяющих всю стратегию сооружения. Модель даёт количественное обоснование.
**Почему это сработает:** Прямое применение всех компонентов модели к стратегическому решению, которое сейчас принимается на основе экспертных суждений.

### А4. Парето-фронт «сроки — стоимость — риск — качество» для заказчика
**Где:** Мониторинг финансовых рисков (Рекомендации, С3); оценка стоимости от класса 5 до класса 3 (G2)
**Что:** Многокритериальная оптимизация: вектор F(Γ) = {T, C, R, Q} → min. Визуализация Парето-фронта для заказчика. При итеративном уточнении стоимости перестраивать фронт, сужая неопределённость.
**Зачем:** Заказчик получает не одну точку, а карту компромиссов для обоснованного выбора баланса сроков/стоимости/риска.
**Почему это сработает:** Многокритериальная оптимизация на графах с атрибутами (раздел 10) спроектирована для задач именно этого класса.

### А5. Детектор рассинхронизации уровней графиков (3/4)
**Где:** Синхронизация графиков 3 и 4 уровней (Рекомендации, G12, G13)
**Что:** Модель как «детектор рассинхронизации»: прогнозировать, где график 4 уровня оторвётся от графика 3 уровня, и заранее перестраивать последовательность работ.
**Зачем:** Повысить управляемость, сократить фиктивную плановую отчётность и реальные просрочки.
**Почему это сработает:** Потеря связи нижнего уровня с верхним становится статистически заметной раньше, чем проявляется в фактическом срыве.

### А6. Обратное планирование от целевой даты пуска через этапы ПНР
**Где:** Планирование «от обратного» (Рекомендации, G6, D7; Construction.md, этапы А–Г)
**Что:** Построить вероятностный граф в обратном направлении: от коммерческой эксплуатации к А-0. Определить необходимые даты готовности систем, завершения монтажа, поставок, FAT/SAT и выпуска РД.
**Зачем:** Сократить риск ситуации, когда стройка формально завершает объекты, но блок не готов к вводу.
**Почему это сработает:** Ввод — не конец линейной цепочки, а набор жёстких условий готовности; обратное планирование хорошо работает в вероятностной сетевой модели.

## Группа Б. ТЕПЛОМОНТАЖНЫЕ РАБОТЫ И КРИТИЧЕСКИЙ ПУТЬ

### Б1. GERT-модель сварки трубопроводов с петлями переделок
**Где:** ТМР — сварка трубопроводов, критический путь (Construction.md, основной период)
**Что:** GERT-модель с вероятностными переходами: сварка → NDE-контроль → годен / ремонт / перерезка. Каждый узел — дерево решений A_k = (µ_k, C_k, R_k). Оптимизация выбора технологии сварки, последовательности монтажа, стратегии контроля и распределения бригад между объектами.
**Зачем:** ТМР — критический путь. Каждый дефект — петля переделки. GERT моделирует петли, оптимизация минимизирует суммарное время. Потенциал сокращения на недели-месяцы.
**Почему это сработает:** GERT с обратными дугами спроектирован для процессов с повторными циклами. Статистика по дефектности сварных соединений накоплена.

### Б2. Динамическая оптимизация критического пути в ходе строительства
**Где:** Основной период — ТМР, электромонтаж, строительные работы (Construction.md; Рекомендации, G)
**Что:** Динамическое перепланирование: при задержке поставок начать с другого участка; упреждающее перераспределение ресурсов на «угрожающие» субкритические пути при миграции критического пути (ТМР → электромонтаж → строительные работы).
**Зачем:** Сокращение основного периода на 3–8 мес. за счёт проактивного управления.
**Почему это сработает:** В ходе стройки критический путь мигрирует. Вероятностная модель предскажет смену критичности заранее.

## Группа В. ЛИЦЕНЗИРОВАНИЕ

### В1. GERT-модель процесса лицензирования с циклами экспертиз
**Где:** Лицензирование (Рекомендации, B3, B5; Приложение 1, раздел 6/18)
**Что:** GERT-сеть с decision trees на узлах: подача заявки → экспертиза → замечания → доработки → повторная подача. ML-классификация рисков замечаний по историческим данным. Монте-Карло для сроков. Оптимизация параллельности: какие пакеты подавать раньше, какие дробить, какие объединять. Критерии: Вальд для safety, Байес для ожидаемых, Гурвиц для баланса.
**Зачем:** Сократить 1–3 года задержек лицензирования. Реалистичный график получения лицензий. Предотвращение фальстарта мобилизации.
**Почему это сработает:** Стандартные ацикличные CPM не работают с петлями обратной связи, которые являются нормой в лицензировании. GERT изначально для этого создан.

### В2. ML-прогноз длительности лицензирования на ранней стадии
**Где:** Адаптация проекта под площадку и национальные требования (Рекомендации, B2); прелицензионные консультации
**Что:** ML-классификатор (XGBoost/CatBoost) обучается на данных прошлых проектов: тип площадки, страна, требования регулятора → прогноз длительности лицензионного процесса и ключевых факторов риска. Результат — входные данные для PERT-модели подготовительного периода.
**Зачем:** Длительность лицензирования варьируется от 0.5 до 6 лет — ключевой источник неопределённости. Прогнозирование на ранней стадии критично для всего графика.
**Почему это сработает:** Данные по десяткам проектов (МАГАТЭ, NEA) позволяют обучить модель. Даже грубый прогноз лучше детерминированной оценки.

## Группа Г. ИНВЕСТИЦИОННАЯ СТРАТЕГИЯ И ФИНАНСЫ

### Г1. Оптимизация объёма ранних инвестиций в подготовку (до FID)
**Где:** Стратегия инвестирования в подготовительный период (Рекомендации, С2)
**Что:** Формализовать как задачу принятия решений: альтернативы Γ — стратегии распределения инвестиций; состояния природы S — сценарии (проект продолжается / отменяется / задержка). Utility u(Γ_i, s_j) — NPV. Критерий Гурвица с α, зависящим от бизнес-модели. Критерий Сэвиджа (minimax regret) для минимизации потерь от неоптимального выбора.
**Зачем:** Математическое обоснование фундаментальной дилеммы: больше подготовки снижает строительные риски, но увеличивает sunk cost. Примеры: Wylfa, VC Summer, Балтийская АЭС.
**Почему это сработает:** Матрица полезности переводит корпоративную дилемму в строгий расчёт ожидаемого ущерба и упущенной выгоды.

### Г2. Стресс-тестирование финансовой модели и расчёт резервов
**Где:** Мониторинг финансовых рисков (Рекомендации, С3)
**Что:** Совместное моделирование рисков графика и стоимости с использованием копула-функций и Монте-Карло. Совместное распределение сроков и затрат, квантили для резервов. Критерии Сэвиджа/Вальда для выбора стратегии реагирования.
**Зачем:** Объективный управленческий резерв бюджета (P80–P90) вместо экспертного «% от сметы». Копулы учитывают корреляцию между рисками.
**Почему это сработает:** Монте-Карло даёт точный размер резерва для заданного уровня уверенности.

### Г3. Оптимизация графика авансирования и cash-flow
**Где:** Финансирование подготовительного периода (Рекомендации, C2; Приложение 1, разделы 11, 23)
**Что:** Стохастическое моделирование для оптимизации графика авансирования с учётом рисков кассовых разрывов и стоимости капитала.
**Зачем:** Снижение стоимости финансирования на 10–15%, бесперебойное финансирование работ.
**Почему это сработает:** Учёт вероятностного характера поступлений и платежей позволяет оптимизировать оборотный капитал.

## Группа Д. КОНТРАКТАЦИЯ ОБОРУДОВАНИЯ И ЦЕПОЧКА ПОСТАВОК

### Д1. Стохастическая оптимизация поставок ОДЦИ (just-in-time)
**Где:** Контрактация ОДЦИ (Рекомендации, J1, G5; Приложение 1, раздел 9)
**Что:** Стохастическая модель графика поставок: каждая единица — дуга с вероятностным временем. ML (XGBoost, GNN) обучается на данных о задержках поставщиков. Монте-Карло для вероятности задержки относительно потребности монтажа. Оптимизация момента заказа: баланс «заморозка средств / хранение» vs. «простой монтажа».
**Зачем:** Несвоевременная поставка ОДЦИ — ключевой риск. Модель оптимизирует just-in-time с учётом неопределённости. Экономия 10–20% на хранении/финансировании.
**Почему это сработает:** ML на исторических данных повышает точность прогноза по сравнению с экспертными оценками. GNN улавливает скрытые паттерны задержек в сетях субподрядчиков.

### Д2. Предиктивный мониторинг 6 волн контрактации (аномалии)
**Где:** Этапы контрактации (Приложение 1, раздел 9.8; Рекомендации, J1–J4)
**Что:** GNN и алгоритмы поиска аномалий на многомерном графе событий. При обнаружении высокого риска срыва сроков — глобальная оптимизация для перестройки логистики или замены поставщика.
**Зачем:** Исключение каскадных задержек на критическом пути.
**Почему это сработает:** GNN улавливают паттерны задержек за месяцы до официального уведомления о срыве.

### Д3. Синхронизация графиков РД, поставок и СМР
**Где:** Графики РД, поставок и СМР (Рекомендации, G4, G5; Приложение 1, раздел 9.6)
**Что:** Интегрированная сетевая модель для автоматического выявления коллизий между графиками выпуска РД, поставок оборудования и начала СМР.
**Зачем:** Снижение простоев из-за отсутствия РД или оборудования на 50%, повышение ритмичности.
**Почему это сработает:** Сквозная прослеживаемость зависимостей между процессами; сигнал о конфликтах заранее.

### Д4. Поставки электрооборудования и АСУ ТП
**Где:** Электротехническое оборудование и АСУ ТП (Рекомендации, G9, D11; Приложение 1, раздел 21.7)
**Что:** ML-прогноз сроков поставки по историческим данным. Монте-Карло для оптимального момента заказа с учётом стоимости хранения и риска дефицита.
**Зачем:** Повышение точности прогноза на 30%, снижение простоев электромонтажных бригад.
**Почему это сработает:** ML выявляет скрытые закономерности в данных по задержкам.

## Группа Е. ПУСКОНАЛАДОЧНЫЕ РАБОТЫ (ПНР)

### Е1. GERT-модель этапов ПНР с петлями возврата
**Где:** Этапы А-0 — А-4, Б, В, Г (Construction.md; Рекомендации, G6)
**Что:** GERT с вероятностными переходами на каждом подэтапе: успех / доработка / повтор. Критерий Вальда (максимин) для безопасных сроков — ПНР связаны с ядерной безопасностью. Байесовские сети для обновления вероятностей по мере поступления результатов.
**Зачем:** Обоснованные резервы для каждого подэтапа. Реалистичная дата физпуска. Сокращение этапа А на 10–15%.
**Почему это сработает:** Вальд применяется для safety-critical operations (раздел 3 статьи). Байесовское обновление даёт адаптивный прогноз.

### Е2. GERT + детекция аномалий на ХГО реакторной установки (А-3)
**Где:** Подэтап А-3: холодная (А-3.1) и горячая (А-3.2) фазы обкатки (Construction.md)
**Что:** GERT: гидроиспытания → годен/не годен; промывка → норма/повтор; горячая обкатка → ОК/доработка. Автоэнкодер на потоковых данных (давление, температура, расход, вибрация ГЦНА) для раннего обнаружения отклонений до официального провала испытания.
**Зачем:** Каждый неуспешный цикл ХГО — недели задержки. Раннее обнаружение отклонений позволяет скорректировать раньше.
**Почему это сработает:** Автоэнкодер на данных обкатки из референтных проектов — стандартная задача предиктивной аналитики в промышленности.

## Группа Ж. УПРАВЛЕНИЕ ИЗМЕНЕНИЯМИ И ЗРЕЛОСТЬ ПРОЕКТА

### Ж1. Мгновенная оценка каскадного влияния проектных изменений
**Где:** Управление изменениями (Рекомендации, D3; Приложение 1, раздел 4.3)
**Что:** При каждом change request: пересчёт модели (обновить дуги → Монте-Карло → новый критический путь → влияние на P80). ML-классификатор (на базе GNN) присваивает степень риска по паттернам прошлых проектов. Многокритериальная оптимизация для выбора оптимального набора изменений.
**Зачем:** Мгновенная количественная оценка последствий. Обоснованное принятие/отклонение запросов. Блокировка цепных реакций задержек. Снижение необоснованных изменений на 40%.
**Почему это сработает:** ML на данных прошлых проектов + быстрый пересчёт Монте-Карло дают объективную математическую базу для споров о стоимости «добровольных улучшений».

### Ж2. Расчёт «порога допустимого первого бетона» по зрелости проекта
**Где:** Зрелость проекта до ПБ (Рекомендации, D1); заморозка требований (D9)
**Что:** ML-модель: % готовности РД → прогноз количества изменений/переделок. Оптимизация: при каком уровне готовности переход к строительству выгоднее, чем дополнительная задержка. Ансамбли ML (Random Forest / CatBoost) для агрегации сотен метрик проекта → классификация «Готов / Риск / Не готов».
**Зачем:** Переход от субъективной оценки к вероятностной. Исключение фатальных перерасходов из-за политически мотивированного старта с «сырым» проектом (Flamanville, Olkiluoto).
**Почему это сработает:** ML, обученный на базе Lessons Learned, лишён когнитивного оптимизма. Data-driven «светофор» для решения о переходе.

## Группа З. КОНТРАКТНАЯ СТРАТЕГИЯ

### З1. Формализация выбора контрактной модели и ценообразования
**Где:** Модель контрактации (Рекомендации, I1); схема ценообразования (I2); распределение рисков (I4)
**Что:** Альтернативы Γ — контрактные модели (EPC-turnkey, split-package, alliance) и схемы ценообразования. Состояния природы — сценарии (задержки, инфляция, валюта, дефицит рабочей силы). Критерий Сэвиджа для минимизации regret. Критерий Гурвица для баланса оптимизма/пессимизма при распределении рисков.
**Зачем:** Количественное сравнение стратегий через utility при различных сценариях, а не качественные оценки.
**Почему это сработает:** Сэвидж предназначен для ситуаций с ограниченной информацией и множеством альтернатив — точное описание выбора контрактной модели.

### З2. Многокритериальный выбор подрядчиков (Парето-оптимальность)
**Где:** Управление рисками / контрактная стратегия (Рекомендации, H; Приложение 1, раздел 17)
**Что:** Многокритериальная оптимизация в пространстве: цена, срок, риск срыва, качество, RPN-индекс поставщика. Парето-фронт вместо скаляризации до минимальной цены.
**Зачем:** Минимизация совокупного риска проекта, а не локальной стоимости договора.
**Почему это сработает:** Модель позволяет найти Парето-оптимальные решения по вектору критериев.

## Группа И. ПОДГОТОВКА ПЛОЩАДКИ И ИНФРАСТРУКТУРА

### И1. Оптимизация последовательности возведения ВЗиС и СМБ
**Где:** ВЗиС, пионерная база, СМБ (Приложение 1, разделы 13, 25; Рекомендации, K2)
**Что:** Локальная оптимизация + ML-кластеризация для пространственно-временного планирования размещения объектов 1–4 пусковых комплексов СМБ. Целевая функция: минимизация времени до готовности критических объектов (БРХ, арматурное хозяйство, энергоснабжение) при ограничениях на ресурсы. Динамическое программирование для расстановки приоритетов.
**Зачем:** Сокращение подготовительного периода, минимизация транспортных плеч, максимальная пропускная способность.
**Почему это сработает:** Размещение десятков объектов — миллионы комбинаций. Алгоритмическая оптимизация vs. экспертное решение.

### И2. Стохастическое моделирование водопонижения и укрепления грунтов
**Где:** Водопонижение и укрепление грунтов (Construction.md, этап 2; Приложение 1, раздел 12.4)
**Что:** Стохастическое моделирование гидрогеологических условий, ML-прогноз по данным грунтов/погоды; оптимизация системы водопонижения с учётом сезонных колебаний.
**Зачем:** Снижение рисков подтопления котлованов, оптимизация затрат на водопонижение.
**Почему это сработает:** Учёт вероятностного характера гидрогеологических данных + альтернативные технологии для площадок с высоким ГВУ.

### И3. Расширение модели на внешнюю инфраструктуру (СВМ, ЛЭП, водоснабжение)
**Где:** Подключение к энергосистеме, СВМ (Рекомендации, A4; Construction.md — объекты СВМ)
**Что:** Расширить модель: «строим блок + выдачу мощности + внешнюю инфраструктуру» (ЛЭП, подстанции, КРУЭ, техводоснабжение, внешние сети).
**Зачем:** Убрать ложную оптимизацию, когда площадка готова, а внешний контур не позволяет выдать мощность.
**Почему это сработает:** Внешняя инфраструктура часто управляется отдельно, хотя влияет на общий срок не меньше работ внутри ограждения.

## Группа К. СТЕПЕНЬ УКРУПНЕНИЯ И ИНДУСТРИАЛЬНЫЕ МЕТОДЫ

### К1. Оптимизация выбора метода монтажа (укрупнение vs. поэлементный)
**Где:** Укрупнение и индустриальные методы (Рекомендации, D4, D5; Construction.md — работы на СМБ)
**Что:** Парето-фронт в пространстве: стоимость аренды крана, скорость, погодные риски, затраты на полигоны укрупнительной сборки СМБ. Оптимизация: какие узлы собирать укрупнённо, где делать предмонтаж, какой размер блоков оптимален для конкретной логистики.
**Зачем:** Обоснованный выбор для конкретной площадки. Сокращение основного периода, снижение трудоёмкости, улучшение качества.
**Почему это сработает:** Выбор степени укрупнения не универсален. Модель доказывает количественно, окупятся ли инвестиции в суперкран и цеха за счёт экономии времени.

## Группа Л. СЕРИЙНОСТЬ И ОБУЧЕНИЕ МЕЖДУ БЛОКАМИ

### Л1. ML-модель эффекта обучения между блоками (learning curve)
**Где:** Серийность и стандартизация (Рекомендации, O1)
**Что:** ML обучается на данных 1-го блока: кластеризация паттернов задержек (K-means, DBSCAN), коррекция оценок для 2-го блока по кривой обучения. Детектор аномалий (Isolation Forest, автоэнкодер) сигнализирует об отклонениях на 2-м блоке. Transfer learning между блоками. Оптимизация шага ввода (12/18/24 мес.) с учётом эффекта масштаба.
**Зачем:** Формализация и отслеживание эффекта обучения. Прогноз реалистичного ускорения. 15–20% экономии на 2+ блоке.
**Почему это сработает:** Максимальная однородность данных: тот же проект, те же подрядчики, та же площадка → высокая предсказательная сила ML.

## Группа М. ПЕРСОНАЛ И СОЦИАЛЬНАЯ ИНФРАСТРУКТУРА

### М1. Resource leveling и оптимизация мобилизации персонала
**Где:** График потребности персонала (Рекомендации, L3; Приложение 1, раздел 14.6)
**Что:** Сдвиг некритичных работ в пределах резервов + ML-прогноз реальной выработки → минимизация пиковой численности. Помесячный прогноз потребности по специальностям с увязкой к жилью, транспорту, столовым, КПП.
**Зачем:** Снижение капзатрат на жилые городки, столовые и КПП, вместимость которых диктуется пиком.
**Почему это сработает:** Алгоритмы «размазывают» искусственные пики без сдвига даты пуска.

### М2. Оптимизация социально-бытовой инфраструктуры
**Где:** СБИ (Приложение 1, раздел 14; Рекомендации, L4)
**Что:** ML (кластеризация/регрессия) для прогноза динамики численности. Многокритериальная оптимизация графика строительства жилых объектов и соцкультбыта.
**Зачем:** Снижение затрат на СБИ на 15–20%, обеспечение 50% персонала за год до ПБ.
**Почему это сработает:** Данные предыдущих проектов позволяют точнее планировать потребность.

## Группа Н. ПРОЕКТИРОВАНИЕ И ЦИФРОВЫЕ ТЕХНОЛОГИИ

### Н1. Интеграция 3D/4D-модели (BIM) с календарно-сетевой моделью
**Где:** PLM/BIM/цифровые двойники (Рекомендации, M1, D2); PIM/K-PIM (F1)
**Что:** Объединение 3D/4D-модели с КСМ: поиск не только геометрических, но и временных коллизий (наложение фронтов, конфликт доступа, блокировка монтажных зон). ML для прогноза «точек будущих коллизий». GNN на 3D + event-graph. Интеграция данных PIM в динамический граф.
**Зачем:** Снижение rework на 20–40%, улучшение координации проектировщиков и строителей.
**Почему это сработает:** Здесь встречаются все шесть компонентов модели: сеть работ, стохастика, геометрия, выбор сценариев, ML и многокритериальная оценка.

### Н2. Предиктивный график сдачи помещений (GNN)
**Где:** График передачи помещений (Рекомендации, G10)
**Что:** Многомерный граф: каждое состояние x_i = (t_i, a_i) с вектором атрибутов (% готовности по видам работ). GNN (Graph Attention Network) обучается на данных референтных проектов. Прогноз реалистичных сроков готовности с учётом зависимостей между смежными помещениями.
**Зачем:** Координация строителей и монтажников. Устранение простоев и работ «поверх» незавершённого строительства.
**Почему это сработает:** GNN спроектированы для данных с топологией — здания АЭС имеют чёткую структуру зависимостей.

## Группа О. УПРАВЛЕНИЕ РИСКАМИ И ИЗВЛЕЧЁННЫЕ УРОКИ

### О1. Формализация реестра рисков как сценарного графа
**Где:** Интегрированная система управления рисками (Рекомендации, H1)
**Что:** Каждый риск — узел с вероятностью, влиянием на T/C/Q и набором мер реагирования. Оптимизация портфеля мер снижения рисков.
**Зачем:** Убрать «слепые зоны», перейти от хранения реестра к оптимизационным решениям.
**Почему это сработает:** Интегрированная модель превращает реестр рисков в инструмент принятия решений.

### О2. ML на базе извлечённых уроков (Lessons Learned)
**Где:** Управление знаниями (Рекомендации, F2, F5)
**Что:** NLP для автоматического сопоставления новых ситуаций с похожими уроками из базы. Кластеризация паттернов проблем, повторяющихся между проектами. Дерево решений: стоит ли применять решение, каковы вероятные последствия.
**Зачем:** Снижение повторяемости ошибок на 40%, масштабирование процесса передачи знаний.
**Почему это сработает:** NLP + ML позволяют обрабатывать большие объёмы неструктурированных данных.

## Группа П. КАЧЕСТВО И МЕТРОЛОГИЯ

### П1. Предиктивная модель качества сварки

**Что:** ML-классификация: квалификация сварщиков, погода, тип материалов → прогноз % брака. Раннее выявление факторов риска → коррекция обучения или технологии.
**Зачем:** Снижение переделок, повышение качества, сокращение затрат на дефекты.
**Почему это сработает:** ML выявляет скрытые корреляции, неочевидные для человека.

## Группа Р. ЭКСПОРТНЫЕ ПРОЕКТЫ И ГЕОПОЛИТИЧЕСКИЕ РИСКИ

### Р1. Сценарный анализ санкций и экспортного контроля
**Где:** Импортные ограничения (Рекомендации, J4); управление изменениями (I6)
**Что:** Модель для сценариев: какие узлы критичны к замене поставщика, где потребуется перепроектирование, какие альтернативы минимизируют regret. NLP для мониторинга новостного фона.
**Зачем:** Уменьшение многолетних задержек из-за вынужденной замены компонентов.
**Почему это сработает:** Minimax regret и сценарный анализ — классика для неполной информации.

### Р2. Оптимизация стратегии локализации
**Где:** Локализация (Рекомендации, D10, J3; Приложение 1, раздел 15.3)
**Что:** Байесовский критерий + многокритериальная оптимизация: вероятности брака и задержек местных поставщиков vs. экономия на логистике и политические требования. ML-классификация потенциала поставщиков.
**Зачем:** Оптимальная стратегия локализации без контрактных споров.
**Почему это сработает:** Если матожидание ущерба от риска срыва превышает выгоду — модель обоснует выбор.

### Р3. Оптимизация интегрированного предложения для экспорта
**Где:** Предконтрактная стадия экспортных проектов (Приложение 1, раздел 16)
**Что:** Многокритериальная оптимизация конфигурации предложения (состав работ, локализация, ценообразование) с учётом валютных/политических рисков. Критерий Гурвица.
**Зачем:** Повышение конкурентоспособности предложения, снижение вероятности убытков.
**Почему это сработает:** Критерии принятия решений — формальный аппарат для многовариантного выбора.

**Зачем:** Повышение надёжности, снижение затрат на внеплановые ремонты.
**Почему это сработает:** ML выявляет зависимости между режимами и деградацией.
